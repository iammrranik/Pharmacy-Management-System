POSTMAN GUIDE - Spring Security + JWT Auth
===========================================

The application now uses JWT token-based authentication.
Almost all endpoints (except Public ones) require a valid token
in the Authorization header.


1. REGISTER A NEW USER (Public - no token needed)
=================================================

Public registration only allows the CUSTOMER role.
ADMIN users must be created by an existing ADMIN via POST /api/users.

POST http://localhost:8080/api/auth/register
Content-Type: application/json

{
    "name": "John Doe",
    "email": "john@test.com",
    "username": "john",
    "password": "pass123",
    "phone": "01711111111",
    "role": "CUSTOMER"
}

Response (201 Created):
{
    "message": "User registered successfully",
    "token": "eyJhbGciOiJIUzI1NiJ9...",
    "data": { ... }
}

Copy the "token" value from the response.

To create an ADMIN user, see step 7 below (bootstrap flow).


2. LOGIN (Public - no token needed)
====================================

POST http://localhost:8080/api/auth/login
Content-Type: application/json

{
    "username": "john",
    "password": "pass123"
}

Response (200 OK):
{
    "token": "eyJhbGciOiJIUzI1NiJ9...",
    "username": "john",
    "role": "CUSTOMER"
}

Copy the "token" value.


3. HOW TO USE THE TOKEN IN POSTMAN
===================================

For every protected endpoint, add a header:

    Key:   Authorization
    Value: Bearer <your_token_here>

Example:
    Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...

In Postman:
  - Go to the "Headers" tab
  - Add "Authorization" as the key
  - Add "Bearer <token>" as the value
  - Or use the "Authorization" tab -> Type = "Bearer Token" -> paste token

If you don't send the token, you'll get:
    401 Unauthorized
    {
        "error": "Unauthorized",
        "status": 401
    }

If you send an invalid/expired token, you'll get:
    403 Forbidden

5. TESTING STEPS IN POSTMAN
============================

Step 1: Register/Login to get a token
  - POST /api/auth/register (create user)
  - OR POST /api/auth/login (get token for existing user)
  - Copy the "token" from the response

Step 2: Set the token as a variable (recommended)
  - In Postman, create a collection variable "token"
  - Set it to the token value
  - Then for all endpoints, set header:
        Authorization: Bearer {{token}}

Step 3: Test Public endpoints (no token needed)
  - GET /api/medicines  -> should work without any header

Step 4: Test ADMIN / CUSTOMER endpoints
  - GET /api/orders  -> add Authorization header with CUSTOMER token -> works
  - GET /api/orders  -> without Authorization header -> 401 Unauthorized

Step 5: Test ADMIN only endpoints
  - POST /api/medicines with ADMIN token -> 201 Created
  - POST /api/medicines with CUSTOMER token -> 403 Forbidden
  - POST /api/medicines without token -> 401 Unauthorized

Step 6: Role switching
  - Register one CUSTOMER via POST /api/auth/register
  - Create an ADMIN via the bootstrap flow (see step 7 below)
  - Login with each to get their tokens
  - ADMIN token can access everything
  - CUSTOMER token gets 403 on ADMIN-only endpoints

Step 7: Getting an ADMIN token (bootstrap)
  - Public /api/auth/register blocks ADMIN role (403)
  - You need one bootstrap ADMIN in the DB first:
    Run the BCrypt INSERT from db.sql, OR use the SQL in api3.txt
  - Login as that bootstrap ADMIN, then create more admins
    via POST /api/users (see api3.txt Scenario 3 for full walkthrough)
